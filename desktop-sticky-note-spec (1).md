# 桌面便签应用 · Desktop Sticky Note App

## 项目概述 Overview

一个运行于浏览器的桌面便签应用，左侧侧边栏含迷你日历 + 两种固定分类（便签 / 待办事项），右侧主区域随分类切换显示对应内容。支持自定义命名、自选背景色、删除、富文本编辑与 To-Do 管理，数据本地持久化。

---

## 技术栈 Tech Stack

- **框架**: 纯 HTML + CSS + Vanilla JS（单文件 `index.html`，无需构建，双击即运行）
- **持久化**: `localStorage`
- **可选桌面化**: Electron（`electron-main.js` 包裹 `index.html`）
- **拖拽排序**: SortableJS（CDN 引入，用于 To-Do 条目排序）

## 功能需求 Feature Requirements

### 1. 侧边栏 Sidebar

#### 1-A 迷你日历 Mini Calendar

- 位置：侧边栏顶部
- 尺寸：宽 `144px`，紧凑排布，约 `7列 × 6行`
- 功能：
  - 显示当前月份，左右箭头切换月份
  - 今天日期高亮（圆形填充色）
  - 有待办/便签的日期显示小圆点标记
  - 点击某日期，主区域筛选显示该日期创建的内容
- 不需要弹出，直接内嵌于侧边栏

#### 1-B 分类列表 Category List（固定两种）

侧边栏日历下方，**只有两个固定分类**，不可新增/删除分类：

| 分类 | 图标 | 说明 |
|------|------|------|
| 📝 便签 | 📝 | 下方展开该分类下所有便签条目 |
| ✅ 待办事项 | ✅ | 下方展开该分类下所有待办列表条目 |

- 每个分类标题可点击**折叠/展开**子列表
- 子列表每条显示：**色块 + 自定义名称 + 删除按钮（×）**
- 点击子条目 → 右侧主区域切换到对应内容
- 当前激活条目高亮
- 每个分类标题右侧有 **[+]** 按钮，点击新建该分类下的条目

---

### 2. 顶部工具栏 Toolbar

工具栏左侧显示当前选中条目的名称（可点击重命名）；右侧操作按钮：

| 按钮 | 功能 |
|------|------|
| 🔍 | 全文搜索（便签内容 / 待办文字） |
| ＋ | 新建当前分类下的条目 |
| 📌 | 置顶当前条目（侧边栏列表排最前） |
| 🎨 | 打开颜色选择器，为当前条目**手动选色** |
| 🗑 | 删除当前条目（弹出确认对话框） |
| ☰ | 菜单（导出 `.txt`/`.md`，快捷键说明） |

---

### 3. 便签主区域 Note Area

当侧边栏选中「便签」分类下某一条目时，右侧显示：

- **背景色**：该便签的自定义颜色（用户通过 🎨 选色器选定，非随机）
- **编辑区**：`contenteditable` div
  - 单击进入编辑
  - 空白时显示占位符：「单击进入编辑，右键可插入图片或复制后粘贴」
  - 自动保存（输入停止 `500ms` 后写入 `localStorage`）
- **右键菜单**：
  - 插入图片（`<input type="file" accept="image/*">`，读取后转 Base64 内联）
  - 粘贴图片（监听 `paste` 事件，`clipboardData.items` 处理图片）
- **创建时间戳**：编辑区底部右下角小字显示「创建于 YYYY-MM-DD」

---

### 4. 待办事项主区域 To-Do Area

当侧边栏选中「待办事项」分类下某一条目时，右侧显示该条目的 To-Do 列表：

- **背景色**：该列表的自定义颜色（通过 🎨 选色器选定）
- **待办条目**：
  - 复选框 + 文本内容
  - 勾选后：文字变灰 + 删除线，自动移入「已完成 / 已放弃」折叠区
  - 右键条目 → 弹出菜单：编辑文字 / 标记放弃 / 删除
  - 支持拖拽排序（SortableJS）
- **新增条目**：
  - 列表底部常驻输入框，占位符「+ 添加新待办…」
  - 输入内容后按 `Enter` 确认新增
  - 新增按钮：输入框右侧「✓」按钮，点击也可确认
- **已完成 / 已放弃区**：
  - 默认折叠，标题栏显示数量（如「已完成 / 已放弃 (3)」）
  - 右侧展开箭头按钮，点击展开/折叠
  - 展开后显示已勾选和已标记放弃的条目，可恢复（取消勾选）或永久删除

### 5. 颜色选择器 Color Picker

> **用户自主选色，非随机切换**

点击工具栏 🎨 按钮后：

- 弹出小型颜色面板（浮层，点击外部关闭）
- 提供 **12 个预设色块**（覆盖粉/黄/绿/蓝/紫/橙/白/灰等）
- 同时提供一个 **`<input type="color">`** 原生拾色器，支持任意颜色
- 选色后立即更新当前条目背景色并保存

### 6. 新建 & 命名 Create & Rename

#### 新建便签 / 新建待办列表

- 点击侧边栏分类标题旁的 **[+]** 或工具栏 **＋**
- 弹出小型对话框（Modal）：
  ```
  ┌──────────────────────────┐
  │  新建便签                 │
  │  名称：[_______________] │
  │  颜色：[色块预选…]        │
  │       [取消]  [创建]     │
  └──────────────────────────┘
  ```
  - 名称：文本输入框，默认值「便签 + 序号」或「待办 + 序号」
  - 颜色：6个快速色块，可后续通过 🎨 修改
  - 点击「创建」后：添加到侧边栏列表，主区域切换到新条目

#### 重命名

- 点击工具栏左侧的条目名称 → 变为可编辑输入框 → 失焦或按 `Enter` 保存
- 侧边栏对应条目名称同步更新

---

### 7. 删除 Delete

#### 删除便签 / 待办列表

- 方式一：工具栏点击 🗑 按钮
- 方式二：侧边栏条目右侧「×」按钮
- 均弹出确认对话框：
  ```
  确认删除「便签A」？此操作不可撤销。
  [取消]  [删除]
  ```
- 确认后从 `localStorage` 移除，侧边栏列表更新，主区域切换到同分类第一条（若无则显示空态）

#### 删除待办条目

- 右键待办条目 → 菜单中选「删除」
- 无需确认对话框（单条条目）

## UI 设计规范 Design Spec

### 颜色系统 Color System

| CSS 变量 | 值 | 用途 |
|---------|----|------|
| `--sidebar-bg` | `#F2F2F2` | 侧边栏背景 |
| `--sidebar-width` | `160px` | 侧边栏宽度 |
| `--toolbar-height` | `44px` | 工具栏高度 |
| `--toolbar-bg` | `rgba(255,255,255,0.85)` | 工具栏背景（毛玻璃效果） |
| `--active-highlight` | `rgba(0,0,0,0.08)` | 激活条目背景 |
| `--text-primary` | `#333` | 主文字 |
| `--text-muted` | `#AAA` | 占位符/次要文字 |
| `--done-text` | `#AAA` | 已完成条目文字 |
| `--border` | `#E0E0E0` | 分割线 |

便签/待办列表的背景色由用户通过颜色选择器决定，存储为 `hex` 字符串，CSS 直接使用 `element.style.backgroundColor = note.color`。

### 交互动效 Animations

| 场景 | 动效 |
|------|------|
| 切换主区域内容 | `opacity: 0→1, translateY: 8px→0, 150ms ease` |
| 勾选 Todo | 删除线展开 `200ms ease`，文字颜色渐变 |
| 折叠/展开区块 | `max-height` 过渡 `300ms ease` |
| 颜色面板弹出 | `scale(0.95→1) + opacity 120ms` |
| 确认对话框 | 背景遮罩淡入 `100ms`，卡片弹入 `200ms` |
| 右键菜单 | `scale(0.9→1) 120ms`，从鼠标位置弹出 |
| 侧边栏条目删除 | 高度收缩 `200ms ease` |

## 开发提示 Dev Notes for AI Coding

1. **单文件优先**：`index.html` 内嵌所有 CSS/JS，双击即运行，无需 npm。
2. **颜色选择器实现**：工具栏 🎨 按钮旁用绝对定位浮层，包含 12 个 `<span>` 色块 + 一个 `<input type="color">`；监听 `change` 事件更新当前条目的 `color` 字段并刷新背景。
3. **命名/重命名**：工具栏标题区使用一个 `<span>` 显示名称，点击后替换为 `<input>`，`blur`/`Enter` 后保存并还原为 `<span>`。
4. **删除确认**：用一个全局 `<dialog>` 元素复用，传入条目 id 和名称，确认后执行删除回调。
5. **迷你日历**：纯 JS 生成，按月渲染 `7×6` 格，当天高亮，有数据的日期加 `::after` 小圆点；切月时重新渲染。
6. **图片存储**：插入图片时转 Base64 内联，注意单条 note 不宜超过 ~4MB。
7. **拖拽排序**：引入 SortableJS CDN，`new Sortable(listEl, { animation: 150, onEnd: saveOrder })`。
8. **空态处理**：分类下无条目时，主区域显示「点击左侧 + 新建」引导文字。
9. **数据初始化**：首次加载若 `localStorage` 无数据，写入一条示例便签和一条示例待办，方便用户立刻看到效果。
